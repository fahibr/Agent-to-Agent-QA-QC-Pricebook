STEP 2 — Revise HardwareType / HardwareStandard attributes
============================================================
After Step 1, revise the pricebook (use the Step 1
corrected file when it exists; otherwise use the original
pricebook) as follows:

1. Detect locale from the filename (e.g. en_HK).
2. HardwareType: fill every missing / blank cell with "NULL".
   Do not highlight these NULL fills.
3. HardwareStandard: for each product row, match HardwareType
   by locale + product category code
   (HardwareType.HDW_TYPE_ID <-> HardwareStandard.HDW_TYPE_ID),
   then:
   - fill missing ATTYPE1-4 from ATTR_TYPE1_ID .. ATTR_TYPE4_ID
   - if ATTYPE1-4 already have values, compare them to
     ATTR_TYPE*_ID and correct mismatches
   - copy / correct DHI from HardwareType.SORT_ORDER
   Highlight every HardwareStandard cell that was filled or
   corrected in yellow.

Retain all worksheets. Do not modify the original
pricebook file; write results to the corrected file.

- HardwareType NULL fill count
- HardwareStandard rows updated
- Mapped vs unmapped product category codes
- Filled vs corrected counts for ATTYPE1-4 and DHI
